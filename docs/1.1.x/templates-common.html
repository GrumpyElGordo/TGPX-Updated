Common Template Variables

Table of Contents
Introduction

Many of the script and e-mail templates share the same template values. This document will describe the commonly used template values that are available.

Common Template Variables

    {$config}

    TGPX configuration settings

    {$config.install_url} - TGPX installation URL
    {$config.preview_url} - The thumbnail URL setting
    {$config.from_email} - E-mail Address setting
    {$config.from_email_name} - E-mail Name setting
    {$config.date_format} - Date Format setting
    {$config.time_format} - Time Format setting
    {$config.timezone} - Timezone setting
    {$config.dec_point} - Decimal Point setting
    {$config.thousands_sep} - Thousands Separator setting


    {$gallery}

    The data for a specific gallery

    {$gallery.gallery_id} - The gallery's ID number
    {$gallery.gallery_url} - The gallery URL
    {$gallery.description} - The gallery description
    {$gallery.keywords} - The keywords assigned to the gallery
    {$gallery.thumbnails} - The number of thumbnails that appear on the gallery
    {$gallery.email} - The e-mail address associated with the gallery
    {$gallery.nickname} - The name/nickname associated with the gallery
    {$gallery.weight} - The gallery's assigned weight
    {$gallery.clicks} - The number of clicks the gallery has received
    {$gallery.submit_ip} - The IP address of the gallery submitter
    {$gallery.gallery_ip} - The IP address of the gallery itself
    {$gallery.type} - The type of gallery; either submitted or permanent
    {$gallery.format} - The format of the gallery; either pictures or movies
    {$gallery.status} - The status of the gallery; one of unconfirmed, pending, approved, used, holding or disabled
    {$gallery.date_scanned} - The date and time the gallery was last scanned
    {$gallery.date_added} - The date and time the gallery was added to the database
    {$gallery.date_approved} - The date and time the gallery was approved
    {$gallery.date_scheduled} - The date and time the gallery is scheduled to become eligible for display
    {$gallery.date_displayed} - The date and time the gallery was selected for display
    {$gallery.date_deletion} - The date and time the gallery is scheduled to be deleted
    {$gallery.partner} - The username of the partner account that submitted the gallery
    {$gallery.administrator} - The username of the administrator that approved the gallery
    {$gallery.has_recip} - A flag to indicate if the gallery has a reciprocal link to your site; 1 for yes, 0 for no
    {$gallery.has_preview} - A flag to indicate if the gallery has one or more preview thumbnails; 1 for yes, 0 for no
    {$gallery.times_selected} - The number of times the gallery has been selected for display from approved status
    {$gallery.used_counter} - The number of times the gallery has been used during a build
    {$gallery.build_counter} - The number of builds that have occurred since the gallery was selected for display
    {$gallery.tags} - The tags assigned to the gallery

    In addition to these default values, all of your user defined gallery fields will be available from this template value as well. So, for example, if you create a user defined field with the name icq_number you can access this value using {$gallery.icq_number}


    {$partner}

    The data for a specific partner account

    {$partner.username} - The partner account username
    {$partner.name} - The name/nickname associated with the partner account
    {$partner.email} - The e-mail address associated with the partner account
    {$partner.ip_address} -
    {$partner.date_added} - The date and time the partner account was added
    {$partner.date_last_submit} - The date and time of the last gallery submission from this account
    {$partner.date_start} - The date and time that the partner account becomes eligible to submit galleries
    {$partner.date_end} - The date and time after which the partner account can no longer submit galleries
    {$partner.per_day} - The number of galleries this account can submit per day
    {$partner.weight} - The weight value assigned to all galleries submitted by this account
    {$partner.submitted} - The number of galleries submitted by this account
    {$partner.removed} - The number of galleries that have been deleted by administrators
    {$partner.status} - The partner account status; one of pending, active, or suspended
    {$partner.allow_redirect} - A flag to indicate if URL redirection is allowed for this account; 1 for yes, 0 for no
    {$partner.allow_norecip} - A flag to indicate if this account is not required to have reciprocal links on their galleries; 1 for yes, 0 for no
    {$partner.allow_autoapprove} - A flag to indicate if this account's galleries will be automatically approved; 1 for yes, 0 for no
    {$partner.allow_noconfirm} - A flag to indicate if this account is not required to confirm gallery submissions by e-mail; 1 for yes, 0 for no
    {$partner.allow_blacklist} - A flag to indicate if this account's galleries can have blacklisted data; 1 for yes, 0 for no

    In addition to these default values, all of your user defined partner fields will be available from this template value as well. So, for example, if you create a user defined field with the name icq_number you can access this value using {$partner.icq_number}


    {$pagination}

    This template value contains details on the pagination available for the page being viewed.

    {$pagination.prev} - An indicator of whether there is a previous page of results
    {$pagination.next} - An indicator of whether there is a next page of results
    {$pagination.start} - The starting position of results displayed on this page
    {$pagination.end} - The ending position of results displayed on this page
    {$pagination.total} - The total number of results
